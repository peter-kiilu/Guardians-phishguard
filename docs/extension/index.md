# Chrome Extension Overview

The PhishGuard Chrome Extension (`extension/`) serves as the client-side agent connecting users to the ML backend. It operates invisibly, monitoring in real-time for emerging threats in two crucial attack vectors: standard web browsing and Gmail inbox usage.

## Architecture & Structure

The extension follows Manifest V3 guidelines. 
It consists of several essential parts:

- `manifest.json`: Configuration for the extension and permission management (`activeTab`, `scripting`, `alarms`, etc.).
- `background.js`: A Service Worker that handles URL interception and fetches backend analysis on tab transitions.
- `content.js`: Injects the blocking UI/warning banners on standard web pages.
- `gmail_content.js`: Specifically designed for the Gmail Single Page Application (SPA), intercepting email components via DOM scraping.
- `warning.html` / `styles.css`: The full-page "Phishing Blocked" warning users encounter when a threat is identified.
- `phishing_domains.js`: An autogenerated library containing the latest 101 known phishing domains parsed directly from the ML dataset.

## Dual Protection Modules

1. **[URL Protection](url-protection.md)**: Proactive interception of domain navigations.
2. **[Gmail Protection](gmail-protection.md)**: Real-time scan and flagging of malicious emails via a specialized DOM MutationObserver.
